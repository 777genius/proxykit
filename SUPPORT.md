# Support

## Where to start

If you are evaluating or integrating `proxykit`, start with:

- [README.md](./README.md)
- [live docs](https://777genius.github.io/proxykit/)
- compile-checked examples in package `example_test.go` files

## Best place for different questions

### Usage questions

Use GitHub Discussions or an issue if Discussions are not enabled yet.

Good topics:

- package choice
- architecture fit
- migration questions
- docs gaps

### Bug reports

Open an issue with:

- the package name
- a minimal reproducer
- expected behavior
- actual behavior
- version information

### Security issues

Do not file those publicly first. See [SECURITY.md](./SECURITY.md).

## Support expectations

This is a library project, not a managed service.

That means support is best-effort and works best when reports are:

- small
- reproducible
- specific about expected behavior
