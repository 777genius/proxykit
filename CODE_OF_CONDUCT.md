# Code of Conduct

## Our standard

This project aims to be a respectful and technically serious place to collaborate.

Examples of behavior that contribute to a healthy community:

- discussing technical tradeoffs in good faith
- giving clear and actionable review feedback
- assuming mistakes can be corrected without hostility
- focusing criticism on code, design, and evidence rather than people

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- deliberately hostile or exclusionary language
- bad-faith review behavior or repeated derailment
- publishing private information without permission

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- discussions around releases and documentation

## Enforcement

Project maintainers may remove, edit, or reject contributions and comments that violate this policy.

If you need to report a serious issue, use the contact path in [SECURITY.md](./SECURITY.md) or open a private report through the repository security process when appropriate.
